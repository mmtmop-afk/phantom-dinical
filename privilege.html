USER_PRIVILEGE_LAB

01. FILE_PERMISSION_MATRIX

พื้นฐานสิทธิ์การเข้าถึงไฟล์ในระบบ Linux (Read/Write/Execute)

# โครงสร้าง: [Owner][Group][Others]
chmod 755 - rwxr-xr-x (Default สำหรับ Folder)
chmod 644 - rw-r--r-- (Default สำหรับ File)
chmod 400 - r-------- (Protect Private Key)

02. OWNERSHIP_LOGIC

การจัดการความเป็นเจ้าของเพื่อความปลอดภัยของ Service

sudo chown phantom:admin config.php
sudo chown -R www-data:www-data /var/www/html
# สำหรับ Web Server

03. SUDOERS_SECURITY

การกำหนดสิทธิ์ Root แบบละเอียด (Granular Control)

# อนุญาตให้ใช้เฉพาะ Service Management
phantom ALL=(ALL) /usr/bin/systemctl restart nginx

# จำกัดการรันคำสั่งโดยไม่ต้องใช้รหัสผ่านเฉพาะกลุ่ม admin
%admin ALL=(ALL) NOPASSWD: ALL

04. ADVANCED_BITS

เทคนิค SUID, SGID และ Sticky Bit สำหรับความปลอดภัยระดับสูง

chmod u+s /usr/bin/binary # รันไฟล์ด้วยสิทธิ์เจ้าของ (SUID)
chmod +t /tmp/shared      # ป้องกันคนอื่นลบไฟล์ (Sticky Bit)

05. PRIVILEGE_AUDIT

คำสั่งสำหรับตรวจสอบความผิดปกติของสิทธิ์ในระบบ

sudo -l             # ตรวจสอบสิทธิ์ sudo ของตนเอง
find / -perm -4000  # ค้นหาไฟล์ SUID ที่อาจเป็นช่องโหว่
last                # ดูประวัติการ Login ล่าสุด
cat /etc/group      # ตรวจสอบสมาชิกในกลุ่มสำคัญ (เช่น wheel, sudo)
